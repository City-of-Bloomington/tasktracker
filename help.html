TaskTracker

Introduction

tasktracker is a web application used by City of Bloomington Utilities departments to manage and track work orders.

Requests

The application is based on requests that users can fill in. Each request has a group, type, location, request summary and other details. Based on the group selected, the user may pick one or more employee of the group to assign the request to. The group manager and the assignees will receive an email about the request to take care of. The assignees will mark their contribution to requests through task (see below). You can attach document, images, pdfs to request by clicking on 'Attachment' button.

A request can be cancelled if the request is not valid or needed any more. the closed request can be reopen to add further tasks if needed.

Tasks

Each request can have one or more tasks, each assignee is recommended to write their acheivement to the task and percent completion. When all tasks are completed the request is closed (completed) and no further actions are needed.

Users & Groups

To use this applicaiton, the user must be added to the users list. The users are arranged in groups. The user may have one or more groups. A group is a collection of users who work in one location or managed by certain work type.

Each group need at least one manager from the list of users. The 'Admin' users can add new users, remove users and create new groups. Add group managers to groups.

User Interface (UI)

The UI is divided into a number of tabs (menu) in the top. Here are the options:
  • New Request: to add a new request, or access the most recent requests.
  • Active Requests: this tab will show the most recent active requests
  • Active Tasks: this tab will show the most recent active tasks (tasks that are not completed yet)
  • Unassigned: this tab will show all unassigned active requests.
  • Assigned to Me: this tab will show you the list of active requests (if any) that were assigned to you
  • Assigned to Others: this tab will show all active requests that were assigned to other users. This will help the managers to balance the distribution of tasks among employees.
  • Search: find any request based on request ID, type, group, location, assignment, status and date range.
  • Admin: this link is only available for 'Admin' users where admins can
    • Reports, create stats reports based on report type, request type, assignment and groups for certain period of time: day, week or date range.
    • Add/Edit users by clicking on 'Users'
    • Add/Edit groups by clicking on 'Groups'
    • Add users to groups and remove users from groups by clicking on 'Users' Groups
    • Add manager to a group or remove a manager from a group

Application Workflow

to add a new request, click on the 'New Request' tab, fill in as many fields as you can, you must provide a brief summary of the request, pick a type and a group. You can pick a location, if you need more specific location details add it to 'Other Location'. Due date if the request is needed to be completed by certain date. If the request is related to certain city employee, you can add his/her info by start typing his/her last name in the 'Requester Name' and then select from the list. You may assign the request to certain user by picking the user from the list of users in 'New Assignment'. If you are not sure but to how to assign, leave the assignment to the group managers, they will take care of assignments. You may add other details about the request in the 'Detail' box as the summary field is limited. After you save the request, you can create tasks. You can add documents, images, pdfs by cliking on 'Attachments' button.

When a new request is entered and saved in the system, an email will be sent to the group manager and the employee who have been assigned to this request. The assignee will receive the email with the summary of the request and a link to the request. The user can click on the link that will take him/her to the Login screen (if not already) logged in.

The assignee can show his/her contribution to the request by creating a new task and mark his/her contribution. The task may be completed in multiple steps by setting the percentage of completion and adding the details in the detail box.

If the assignee complete the required task, he/she should mark as 100% completion or click on 'Completed' button. If all tasks are completed (100%), the request status will change to 'Closed' and no further tasks is needed.

The request may be 'Cancelled' if the request is not needed anymore, by click on 'Cancel this request' button on the request page. The closed request can be reopen by clicking on 'Reopen this request' button.
